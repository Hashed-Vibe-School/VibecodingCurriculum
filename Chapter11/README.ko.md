## 왜 필요합니까?

생각해보십시오 - 모든 회사, 프리랜서, 전문가는 웹사이트가 있습니다. 누군가 당신을 온라인에서 검색하면 무엇을 찾을 것입니까?

**포트폴리오 웹사이트가 필요한 실제 상황:**

- **취업 준비**: 채용 담당자는 이력서만 읽는 게 아니라 직접 만든 것을 보고 싶어합니다
- **프리랜서**: 클라이언트는 당신을 고용하기 전에 실력 증명을 원합니다
- **개인 브랜딩**: 수천 명의 다른 지원자들 사이에서 돋보이십시오
- **네트워킹**: 사람들에게 기억에 남을 무언가를 주십시오

> 포트폴리오는 디지털 명함 같지만, 훨씬 더 강력합니다. 당신이 "할 수 있다고 말하는 것"이 아니라 "실제로 할 수 있는 것"을 보여줍니다.

### 쉬운 비유: 포트폴리오는 쇼윈도우

당신이 제빵사라고 상상해보십시오. "맛있는 케이크를 만듭니다"라고 적힌 명함을 돌릴 수도 있고, 아름다운 케이크로 가득 찬 가게 쇼윈도우를 가질 수도 있습니다. 어느 쪽이 사람들이 사고 싶게 만들겠습니까?

포트폴리오 웹사이트는 바로 그 쇼윈도우입니다 - 당신의 최고의 작품을 세상에 전시합니다.

## 단계별 개발

### 1단계: 기본 HTML 만들기

```
> index.html에 기본 구조 만들어줘.
> 헤더, 메인, 푸터 섹션으로 나눠줘.
```

Claude가 이런 구조를 만들어줍니다:

```html
<!DOCTYPE html>
<html lang="ko">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>내 포트폴리오</title>
    <link href="style.css" rel="stylesheet">
</head>
<body>
    <header>
        <!-- 네비게이션 -->
    </header>

    <main>
        <!-- 메인 컨텐츠 -->
    </main>

    <footer>
        <!-- 푸터 -->
    </footer>

    <script src="script.js"></script>
</body>
</html>
```

### 2단계: 자기소개 섹션

```
> 자기소개 섹션 추가해줘.
> - 프로필 사진 자리
> - 이름
> - 한 줄 소개
> - 간단한 자기소개 문단
```

### 3단계: 프로젝트 갤러리

```
> 프로젝트 섹션 추가해줘.
> - 카드 형태로 3개 프로젝트
> - 각 카드에 썸네일, 제목, 설명
> - 호버하면 살짝 올라오는 효과
```

### 4단계: 연락처 폼

```
> 연락처 섹션 추가해줘.
> - 이름 입력
> - 이메일 입력
> - 메시지 입력
> - 보내기 버튼
```

### 5단계: 스타일링

```
> Tailwind CSS 적용해줘.
> 깔끔하고 현대적인 느낌으로.
> 색상은 파란색 계열로.
```

### 6단계: 반응형 만들기

```
> 모바일에서도 잘 보이게 반응형으로 바꿔줘.
> 작은 화면에서는 메뉴가 햄버거로 바뀌게.
```

## 문제가 발생하면

당황하지 마십시오. 자주 발생하는 문제들의 빠른 해결법입니다.

### HTML 파일을 열었는데 아무것도 안 보입니다
- 파일이 `.html` 확장자로 저장되었는지 확인하십시오 (`.txt`가 아니라)
- 파일을 우클릭해서 "연결 프로그램" > 브라우저 선택해보십시오
- 파일이 생각한 폴더에 있는지 확인하십시오

### 페이지가 너무 밋밋합니다 / 스타일이 없습니다
```
> 페이지에 스타일이 없어. Tailwind CSS가 제대로 연결됐는지 확인해줘.
```

### 이미지가 안 나옵니다
- 이미지 파일이 프로젝트 폴더에 있는지 확인하십시오
- 이미지 경로는 대소문자를 구분합니다: `Photo.jpg`와 `photo.jpg`는 다릅니다

### 수정한 게 안 보입니다
- 브라우저 강력 새로고침: Ctrl+Shift+R (Windows) 또는 Cmd+Shift+R (Mac)
- 새로고침 전에 파일을 저장했는지 확인하십시오

### 모바일에서 레이아웃이 이상합니다
```
> 모바일 레이아웃이 깨졌어. 반응형 클래스 확인해줘.
```

## 자주 하는 실수

초보자들이 자주 하는 실수들입니다. 미리 배워두십시오.

### 실수 1: 처음부터 너무 복잡하게 시작하기
**나쁜 접근:**
```
> 애니메이션, 3D 효과, 비디오 배경, 파티클 효과,
> 챗봇이 있는 포트폴리오 만들어줘.
```

**좋은 접근:**
```
> 자기소개, 프로젝트, 연락처 섹션이 있는 간단한 포트폴리오 만들어줘.
```
간단하게 시작하고, 기능은 나중에 추가하십시오.

### 실수 2: 모바일에서 테스트 안 하기
큰 모니터에서는 멋져 보이는 포트폴리오가... 대부분의 사람들은 핸드폰으로 볼 것입니다. 항상 모바일 화면을 확인하십시오.

```
> 이거 모바일에서 어떻게 보이는지 보여줘
```

### 실수 3: 플레이스홀더 콘텐츠 바꾸는 걸 잊기
Claude는 "Lorem ipsum" 같은 더미 텍스트와 예시 이미지를 사용합니다. 공유하기 전에 진짜 내용으로 바꾸는 걸 잊지 마십시오.

### 실수 4: 폰트와 색상을 너무 많이 쓰기
폰트는 1-2개, 색상은 2-3개까지만 쓰십시오. 그보다 많으면 지저분하고 비전문적으로 보입니다.

### 실수 5: 아무 데도 안 가는 링크
모든 링크는 어딘가로 연결되어야 합니다. 공개하기 전에 모든 링크를 테스트하십시오.

```
> 이 프로젝트의 모든 링크 확인해서 끊어진 거 알려줘
```
