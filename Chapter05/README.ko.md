## 왜 필요합니까?

파일만으로는 소프트웨어가 작동하지 않습니다. 도구를 설치하고, 서버를 시작하고, 코드를 테스트하려면 명령어를 실행해야 합니다. 명령어는 파일에 생명을 불어넣는 행동이라고 생각하세요.

**실제 상황:**
- 웹사이트를 만들었고, 이제 브라우저에서 보고 싶다
- 새 기능을 추가하려고 패키지를 설치해야 한다
- 코드가 제대로 작동하는지 테스트하고 싶다
- 코드에 에러가 있어서 디버그해야 한다

### 쉬운 비유: 주방 가전

파일이 재료라면, 터미널 명령어는 주방 가전입니다.

- `npm install` = 장보기 (필요한 재료 구하기)
- `npm run dev` = 불 켜기 (프로젝트 시작하기)
- `npm test` = 맛보기 (잘 됐는지 확인하기)
- `npm run build` = 포장하기 (실제 세상에 내보낼 준비)

가스레인지가 내부적으로 어떻게 작동하는지 알 필요 없습니다. "불을 켜면 요리할 수 있다"만 알면 됩니다. 명령어도 마찬가지입니다. Claude가 세부 사항을 알고, 여러분은 원하는 것만 말하면 됩니다.

## Claude에게 명령어 실행시키기

Claude Code는 터미널 명령어를 대신 실행할 수 있습니다.

### 예시

```
> 이 폴더에 있는 파일 목록 보여줘
```

Claude가 `ls` 명령어를 실행해서 표시합니다.

```
> node 버전 확인해줘
```

Claude가 `node --version`을 실행합니다.

### 승인 절차

명령어를 실행하기 전에 Claude가 확인을 요청합니다:

```
Claude wants to run: ls -la
[Allow] [Deny]
```

`Allow`를 선택하면 실행됩니다.

## 알아두면 좋은 터미널 명령어

직접 외울 필요는 없습니다. "파일 목록 보여줘"라고 하면 Claude가 `ls`를 실행합니다. 그러나 기본 명령어를 알아두면 더 정확하게 요청할 수 있습니다.

### 파일/폴더 관련

| 명령어 | 기능 | 예시 |
|--------|------|------|
| `ls` | 파일 목록 보기 | `ls`, `ls -la` |
| `cd` | 폴더 이동 | `cd src` |
| `mkdir` | 폴더 생성 | `mkdir new-folder` |
| `rm` | 파일 삭제 | `rm file.txt` |
| `cp` | 파일 복사 | `cp a.txt b.txt` |
| `mv` | 파일 이동/이름 변경 | `mv old.txt new.txt` |

### 프로젝트 관련

| 명령어 | 기능 | 예시 |
|--------|------|------|
| `npm install` | 패키지 설치 | `npm install` |
| `npm run dev` | 개발 서버 실행 | `npm run dev` |
| `npm run build` | 빌드 | `npm run build` |
| `npm test` | 테스트 실행 | `npm test` |

### 시스템 관련

| 명령어 | 기능 | 예시 |
|--------|------|------|
| `pwd` | 현재 위치 확인 | `pwd` |
| `cat` | 파일 내용 보기 | `cat package.json` |
| `which` | 프로그램 위치 확인 | `which node` |

## 프로젝트 실행

Claude에게 프로젝트를 실행하라고 요청할 수 있습니다.

### 예시

```
> 이 프로젝트 어떻게 실행해?
```

Claude가 `package.json`을 확인하고 방법을 안내합니다.

```
> 개발 서버 켜줘
```

Claude가 `npm run dev` 같은 명령어를 실행합니다.

```
> 테스트 돌려줘
```

Claude가 `npm test`를 실행하고 결과를 표시합니다.

---

## ! 접두사로 바로 실행

`!`를 붙이면 명령어를 바로 실행하고 결과를 Claude에게 전달합니다.

```
> !npm run build
```

빌드를 실행하고, 결과(성공/실패)를 Claude가 인식합니다.

```
> !git status
```

git 상태를 확인하고, Claude가 그 내용을 인식합니다.

### !와 일반 요청의 차이

| 방식 | 예시 | 차이점 |
|------|------|--------|
| 일반 | `npm install 해줘` | Claude가 명령어 결정, 승인 필요 |
| `!` 접두사 | `!npm install` | 바로 실행, 결과를 Claude가 인식 |

`!`는 실행할 명령어를 정확히 알 때 사용합니다.

## 실습: 간단한 프로젝트 만들기

### 1. 폴더 생성

```
> my-first-project 폴더 만들어줘
```

### 2. 프로젝트 초기화

```
> 방금 만든 폴더에서 npm 프로젝트 초기화해줘
```

Claude가 `npm init -y`를 실행합니다.

### 3. 패키지 설치

```
> express 설치해줘
```

Claude가 `npm install express`를 실행합니다.

### 4. 코드 생성

```
> index.js 만들어줘. 간단한 웹서버로.
> "Hello World"를 보여주는 서버야.
```

### 5. 실행

```
> 서버 실행해줘
```

Claude가 `node index.js`를 실행합니다.

### 6. 확인

브라우저에서 `http://localhost:3000`을 열어 확인합니다.

## 미니 프로젝트: 나만의 API 서버 ⭐

Hello World 서버를 확장해서 간단한 API를 만들어봅니다.

### 목표

- Express 서버 이해
- API 엔드포인트 생성

### 생성

```
> 방금 만든 서버에 API 추가해줘.
> GET /api/hello 하면 { "message": "안녕하세요!" } 응답하는.
```

### 확장

```
> GET /api/time 하면 현재 시간 알려주는 API 추가해줘

> GET /api/random 하면 1-100 사이 랜덤 숫자 주는 API 추가해줘

> GET /api/greeting?name=홍길동 하면 "안녕하세요, 홍길동님!" 응답하게 해줘
```

### 도전 과제

```
> POST /api/todos로 할 일 추가하고
> GET /api/todos로 목록 가져오는 API 만들어줘
> (메모리에 저장해도 돼)
```

## 에러 발생 시

명령어 실행 중 에러가 발생하면, Claude에게 그대로 전달합니다.

```
> 방금 에러 났는데, 왜 그런거야?
```

```
> 에러 고쳐줘
```

**에러 해결 요청 팁:**

```
> 이 에러 메시지가 뭔 뜻이야? [에러 메시지]
> 비슷한 에러가 계속 나와. 근본 원인 찾아줘.
```

에러 메시지를 그대로 붙여넣으면 Claude가 원인과 해결책을 찾아줍니다.

### 자주 발생하는 에러

**"command not found"**
→ 해당 프로그램이 설치되지 않았습니다. Claude에게 설치를 요청하십시오.

**"permission denied"**
→ 권한 문제. `sudo`가 필요할 수 있습니다.

**"ENOENT: no such file or directory"**
→ 파일이나 폴더가 없습니다. 경로를 확인하십시오.

## 위험한 명령어

Claude는 위험한 명령어를 실행하기 전에 경고합니다.

### 주의가 필요한 명령어

| 명령어 | 위험도 | 이유 |

| `chmod 777` | 중간 | 보안 설정 변경 |

Claude가 이런 명령어를 사용하려고 하면 한 번 더 확인하십시오.

## 자주 하는 실수

### 1. 잘못된 폴더에서 명령어 실행하기
`my-project` 폴더에 프로젝트를 만들고 다른 위치에서 명령어를 실행하면 작동하지 않습니다. 항상 위치를 확인하십시오:
```
> 지금 어디야?
```

### 2. 명령어 끝날 때까지 안 기다리기
`npm install` 같은 명령어는 시간이 걸립니다. 명령어가 끝날 때까지 새로운 요청을 입력하지 마십시오.

### 3. 의존성 설치 잊어버리기
Claude가 `package.json`이 있는 프로젝트를 만들면, 프로젝트를 실행하기 전에 `npm install`을 해야 합니다. 안 그러면 "module not found" 에러가 납니다.

### 4. 서버 실행 중에 터미널 닫기
`npm run dev`로 서버를 시작하고 터미널을 닫으면 서버가 멈춥니다. 서버 사용 중에는 터미널을 열어두십시오.

### 5. 에러 메시지 무시하기
에러 메시지는 무섭게 보이지만 유용한 정보가 들어있습니다. 에러를 복사해서 Claude에게 설명해달라고 하십시오.

## 안 되면?

**"command not found" 에러가 발생합니까?**
- 해당 프로그램이 설치되지 않은 것입니다. Claude에게 설치해달라고 하십시오.
- 예시: "node 설치해줘" 또는 "npm 설치해줘"

**"permission denied" 에러가 발생합니까?**
- 관리자 권한이 필요할 수 있습니다. `sudo` 사용에 대해 Claude에게 물어보십시오 (조심히 사용).
- Windows에서는 터미널을 관리자로 실행해보십시오.

**"ENOENT: no such file or directory"가 발생합니까?**
- 존재하지 않는 파일에 접근하려는 것입니다.
- 현재 폴더 위치를 확인하십시오.
- 파일/폴더 이름 철자가 맞는지 확인하십시오.

**서버가 안 켜집니까?**
- 다른 프로그램이 같은 포트를 사용하고 있을 수 있습니다.
- Claude에게: "3000번 포트가 사용 중인 것 같아. 어떻게 해?"

**브라우저에서 변경 사항이 안 보입니까?**
- `Ctrl + F5`로 강력 새로고침 해보십시오
- 서버가 아직 실행 중인지 확인하십시오
- 파일을 저장했는지 확인하십시오

**모든 것이 멈춘 것 같습니까?**
- `Ctrl + C`를 눌러서 현재 작업을 중지하십시오
- `/exit`로 종료하고 `claude`로 다시 시작해보십시오
