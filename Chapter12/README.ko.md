## 왜 필요합니까?

멋진 걸 만들었습니다. 하지만 지금은 오직 당신만 볼 수 있습니다. 맛있는 음식을 만들어 놓고 잠긴 방에서 혼자 먹는 것과 같습니다.

**배포가 필요한 실제 상황:**

- **취업 지원**: "제 포트폴리오입니다"라며 진짜 링크를 보내는 건, "나중에 보여드릴게요"보다 100배 낫습니다
- **클라이언트 프레젠테이션**: 화면 공유 미팅 잡을 필요 없이 바로 작업물 공유
- **협업**: 팀원들이 어디서든 프로젝트를 보고 테스트할 수 있음
- **피드백**: 친구, 멘토, 인터넷 커뮤니티에서 의견 받기

> 배포하기 전까지, 당신의 프로젝트는 바깥 세상에 "존재하지 않습니다".

### 쉬운 비유: 레시피에서 레스토랑으로

컴퓨터에서 웹사이트 만드는 건 집 부엌에서 레시피를 완성하는 것과 같습니다. 배포는 누구나 와서 음식을 맛볼 수 있는 레스토랑을 여는 것입니다.

`localhost:3000` = 당신의 부엌 (당신만 먹을 수 있음)
`yoursite.vercel.app` = 당신의 레스토랑 (누구나 방문 가능)

## 따라해보기: 5분 만에 배포하기

깊이 들어가기 전에, 이게 얼마나 쉬운지 증명해봅시다. Chapter 11에서 만든 프로젝트가 있다면:

```
> git 초기화하고, "my-first-deploy"라는 GitHub 저장소 만들고,
> 모든 파일 푸시해줘. public으로 해줘.
```

그다음 [vercel.com](https://vercel.com)에 가서, "New Project" 클릭, 저장소 선택, "Deploy" 클릭. 끝입니다.

잘 됐다면, 이제 라이브 웹사이트가 생겼습니다. 지금 바로 친구에게 링크를 공유해보십시오. 이것이 바로 배포의 효과입니다.

---

## 배포란 무엇입니까?

배포는 내 컴퓨터에 있는 웹사이트를 인터넷에 올리는 것입니다.

배포하면:
- 누구나 내 웹사이트를 볼 수 있음
- 링크를 공유할 수 있음
- 포트폴리오로 활용 가능

### 배포 전 vs 배포 후

| 상태 | 주소 | 누가 볼 수 있습니까 |
|------|------|-----------------|
| 배포 전 | `http://localhost:3000` | 나만 |
| 배포 후 | `https://my-site.vercel.app` | 전 세계 누구나 |

---

## Vercel: 가장 쉬운 배포

Vercel은 무료로 웹사이트를 배포할 수 있는 서비스입니다.

### 왜 Vercel입니까?

- 무료
- 쉬움 (클릭 몇 번이면 끝)
- 빠름
- HTTPS 자동 적용

### 준비물

1. GitHub 계정
2. Vercel 계정 (GitHub으로 가입)

---

## Vercel 배포하기

### 1단계: GitHub에 코드 올리기

```
> 이 프로젝트 GitHub에 올려줘.
> 저장소 이름은 my-portfolio로.
```

Claude가:
1. Git 초기화
2. GitHub에 저장소 생성
3. 코드 푸시

**배포 요청 팁:**

```
> 이 프로젝트 GitHub에 올려줘. 저장소는 public으로.
> 커밋 메시지는 "Initial commit"으로 해줘.
```

저장소 설정(public/private)이나 커밋 메시지를 지정하면 원하는 대로 배포됩니다.

### 2단계: Vercel 연결

1. [vercel.com](https://vercel.com) 접속
2. GitHub으로 로그인
3. "New Project" 클릭
4. 방금 만든 저장소 선택
5. "Deploy" 클릭

### 3단계: 완료

배포가 끝나면 주소를 받습니다:
```
https://my-portfolio-abc123.vercel.app
```

이 주소로 누구나 내 웹사이트를 볼 수 있습니다.

## 자동 배포

Vercel의 좋은 점: 코드를 수정하면 자동으로 다시 배포됩니다.

### 수정 후 자동 배포 흐름

```
1. 코드 수정
> 제목을 "안녕하세요"로 바꿔줘

2. GitHub에 푸시
> 변경사항 커밋하고 푸시해줘

3. 자동으로 배포됨 (1-2분 소요)

4. 새 버전이 반영됨
```

### 배포 상태 확인

```
> Vercel 배포 상태 확인해줘
```

또는 Vercel 대시보드에서 직접 확인할 수 있습니다.

---

## 커스텀 도메인 연결

기본 주소 (`xxx.vercel.app`) 대신 내 도메인을 쓸 수 있습니다.

### 도메인 구매

- [가비아](https://www.gabia.com)
- [Namecheap](https://www.namecheap.com)

### Vercel에 연결

1. Vercel 대시보드 → Settings → Domains
2. 도메인 입력 (예: `myname.com`)
3. DNS 설정 안내 따라하기

```
> 커스텀 도메인 설정 방법 알려줘
```

## Railway: 백엔드 배포

정적 웹사이트 말고 서버가 필요하면 Railway를 씁니다.

### 언제 Railway를 씁니까?

- 데이터베이스가 필요할 때
- API 서버가 필요할 때
- Node.js/Python 백엔드가 있을 때

### 예시

```
> Express 서버 만들어줘.
> /api/hello 하면 "안녕하세요" 응답하는.
```

---

## Railway 배포하기

### 1단계: Railway 가입

[railway.app](https://railway.app)에서 GitHub으로 가입

### 2단계: 프로젝트 만들기

1. "New Project" 클릭
2. "Deploy from GitHub repo" 선택
3. 저장소 선택
4. 자동으로 배포 시작

### 3단계: 환경 변수 설정 (필요하면)

```
> Railway에 환경 변수 설정하는 법 알려줘
```

비밀번호나 API 키 같은 민감한 정보는 환경 변수로 넣습니다.

## 배포 문제 해결

### 빌드 에러

```
> Vercel 빌드 에러 나왔어: [에러 메시지]
> 뭐가 문제야?
```

### 페이지가 안 뜰 때

```
> 배포는 됐는데 404 에러 나와. 왜 그래?
```

흔한 원인:
- 파일 이름이 `index.html`이 아님
- 경로가 잘못됨
- 빌드 설정이 안 맞음

### 환경 변수 문제

```
> API 키가 작동 안 해. 환경 변수 확인해줘.
```

## 문제가 발생하면

배포는 여러 이유로 실패할 수 있습니다. 빠른 해결법들입니다.

### "Git command not found" 오류
Git이 설치 안 됐습니다. Claude에게 물어보십시오:
```
> 내 컴퓨터에 git 설치하는 법 알려줘
```

### GitHub 푸시 실패 / 인증 오류
- GitHub에 로그인되어 있는지 확인하십시오
- HTTPS를 쓴다면 개인 액세스 토큰이 필요할 수 있습니다
- 이렇게 물어보십시오:
```
> GitHub 인증 설정 도와줘. 푸시 에러가 나와.
```

### Vercel 빌드 실패
가장 흔한 문제입니다. 에러 메시지를 확인하고 물어보십시오:
```
> Vercel 빌드 에러 났어: [에러 메시지 붙여넣기]
> 뭐가 문제고 어떻게 고쳐?
```

### 배포 후 404 페이지가 나옴
- 메인 파일 이름이 `index.html`이 아닐 수 있습니다
- 파일이 하위 폴더에 있을 수 있습니다
```
> 배포된 사이트가 404 떠. 파일 구조 확인해줘.
```

### 사이트는 뜨는데 깨져 보임 (스타일 없음)
- CSS 파일 경로가 잘못됐을 수 있습니다
- 파일 이름 대소문자 확인하십시오
```
> 배포된 사이트에 스타일이 없어. CSS 링크 확인해줘.
```

### 환경 변수가 작동 안 함
- Vercel 대시보드에 추가했습니까?
- 추가 후 다시 배포했습니까?
- 변수 이름은 대소문자를 구분합니다

## 자주 하는 실수

다른 사람들의 실수에서 배우십시오.

### 실수 1: 민감한 정보 커밋하기
**절대 GitHub에 커밋하면 안 되는 것들:**
- API 키
- 비밀번호
- 비밀이 담긴 `.env` 파일

```
> 푸시하기 전에 내 코드에 민감한 정보 있는지 확인해줘
```

실수로 비밀을 커밋했다면, 바로 바꾸십시오 - 이전 값이 git 기록에서 보일 수 있습니다.

### 실수 2: .gitignore에 node_modules 안 넣기
이 폴더는 엄청 크고 절대 커밋하면 안 됩니다:
```
> .gitignore에 node_modules 있는지 확인해줘
```

### 실수 3: 로컬 절대 경로 사용하기
이건 당신 컴퓨터에서만 작동합니다:
```html
<img src="C:/Users/홍길동/project/image.png">
```
배포 후엔 안 됩니다. 상대 경로를 쓰십시오:
```html
<img src="./images/photo.png">
```

### 실수 4: 배포 완료 기다리지 않기
배포는 1-2분 걸립니다. 푸시 직후 확인하면 이전 버전이 보일 수 있습니다. 좀 기다리십시오.

### 실수 5: 잘못된 빌드 설정
프로젝트가 프레임워크(React, Vue 등)를 사용한다면, Vercel에 올바른 빌드 설정이 필요합니다. 보통 자동 감지하지만, 안 되면:
```
> 내 [프레임워크] 프로젝트의 Vercel 빌드 설정 뭘로 해야 해?
```
