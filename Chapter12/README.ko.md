## 배포란?

배포는 내 컴퓨터에 있는 웹사이트를 인터넷에 올리는 것입니다.

배포하면:
- 누구나 내 웹사이트를 볼 수 있음
- 링크를 공유할 수 있음
- 포트폴리오로 활용 가능

### 배포 전 vs 배포 후

| 상태 | 주소 | 누가 볼 수 있나 |
|------|------|-----------------|
| 배포 전 | `http://localhost:3000` | 나만 |
| 배포 후 | `https://my-site.vercel.app` | 전 세계 누구나 |

---

## Vercel: 가장 쉬운 배포

Vercel은 무료로 웹사이트를 배포할 수 있는 서비스입니다.

### 왜 Vercel인가요?

- 무료
- 쉬움 (클릭 몇 번이면 끝)
- 빠름
- HTTPS 자동 적용

### 준비물

1. GitHub 계정
2. Vercel 계정 (GitHub으로 가입)

---

## Vercel 배포하기

### 1단계: GitHub에 코드 올리기

```
> 이 프로젝트 GitHub에 올려줘.
> 저장소 이름은 my-portfolio로.
```

Claude가:
1. Git 초기화
2. GitHub에 저장소 생성
3. 코드 푸시

### 2단계: Vercel 연결

1. [vercel.com](https://vercel.com) 접속
2. GitHub으로 로그인
3. "New Project" 클릭
4. 방금 만든 저장소 선택
5. "Deploy" 클릭

### 3단계: 완료

배포가 끝나면 주소를 받습니다:
```
https://my-portfolio-abc123.vercel.app
```

이 주소로 누구나 내 웹사이트를 볼 수 있습니다.

---

## 자동 배포

Vercel의 좋은 점: 코드를 수정하면 자동으로 다시 배포됩니다.

### 수정 후 자동 배포 흐름

```
1. 코드 수정
> 제목을 "안녕하세요"로 바꿔줘

2. GitHub에 푸시
> 변경사항 커밋하고 푸시해줘

3. 자동으로 배포됨 (1-2분 소요)

4. 새 버전이 반영됨
```

### 배포 상태 확인

```
> Vercel 배포 상태 확인해줘
```

또는 Vercel 대시보드에서 직접 확인할 수 있습니다.

---

## 커스텀 도메인 연결

기본 주소 (`xxx.vercel.app`) 대신 내 도메인을 쓸 수 있습니다.

### 도메인 구매

- [가비아](https://www.gabia.com)
- [Namecheap](https://www.namecheap.com)

### Vercel에 연결

1. Vercel 대시보드 → Settings → Domains
2. 도메인 입력 (예: `myname.com`)
3. DNS 설정 안내 따라하기

```
> 커스텀 도메인 설정 방법 알려줘
```

## Railway: 백엔드 배포

정적 웹사이트 말고 서버가 필요하면 Railway를 씁니다.

### 언제 Railway를 쓸까?

- 데이터베이스가 필요할 때
- API 서버가 필요할 때
- Node.js/Python 백엔드가 있을 때

### 예시

```
> Express 서버 만들어줘.
> /api/hello 하면 "안녕하세요" 응답하는.
```

---

## Railway 배포하기

### 1단계: Railway 가입

[railway.app](https://railway.app)에서 GitHub으로 가입

### 2단계: 프로젝트 만들기

1. "New Project" 클릭
2. "Deploy from GitHub repo" 선택
3. 저장소 선택
4. 자동으로 배포 시작

### 3단계: 환경 변수 설정 (필요하면)

```
> Railway에 환경 변수 설정하는 법 알려줘
```

비밀번호나 API 키 같은 민감한 정보는 환경 변수로 넣습니다.

---

## 무료 플랜 비교

| 서비스 | 무료 제공 | 적합한 용도 |
|--------|-----------|-------------|
| Vercel | 무제한 정적 사이트 | 포트폴리오, 블로그 |
| Railway | 월 $5 크레딧 | 간단한 백엔드 |
| Netlify | 무제한 정적 사이트 | Vercel 대안 |
| Render | 750시간/월 | 백엔드 |

### 처음엔 Vercel로 시작

정적 웹사이트면 Vercel이 가장 쉽고 좋습니다.
백엔드가 필요해지면 그때 Railway 추가하세요.

## 배포 문제 해결

### 빌드 에러

```
> Vercel 빌드 에러 나왔어: [에러 메시지]
> 뭐가 문제야?
```

### 페이지가 안 뜰 때

```
> 배포는 됐는데 404 에러 나와. 왜 그래?
```

흔한 원인:
- 파일 이름이 `index.html`이 아님
- 경로가 잘못됨
- 빌드 설정이 안 맞음

### 환경 변수 문제

```
> API 키가 작동 안 해. 환경 변수 확인해줘.
```
