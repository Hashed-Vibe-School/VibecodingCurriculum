## 들어가기 전에

코딩을 몰라도 괜찮습니다. 이 커리큘럼은 처음부터 시작합니다.

"나는 개발자가 아닌데?"라고 생각할 수 있습니다. 맞습니다, 지금은요. 하지만 AI 시대에는 누구나 자신의 아이디어를 직접 구현할 수 있습니다.

---

## 왜 필요합니까?

이런 상황을 생각해보십시오:

- **"사이드 프로젝트용 간단한 웹사이트를 만들고 싶은데, 개발자 고용하기엔 비용이..."**
- **"회사에서 반복하는 작업을 자동화하고 싶은데..."**
- **"앱 아이디어는 있는데 만들 실력이 없어서..."**
- **"코딩 배우고 싶은데 어디서 시작해야 할지 모르겠어..."**

AI 코딩은 이 모든 것을 해결합니다. 더 이상 수년간 공부할 필요 없습니다. 원하는 것을 AI에게 말하면, AI가 만들어줍니다.

### 쉬운 비유: 통역사

프랑스어로 편지를 쓰고 싶은데, 한국어밖에 못 한다고 상상해보십시오.

**예전 방식**: 프랑스어를 몇 년간 공부하고, 문법 배우고, 어휘 외우고, 작문 연습하고...

**AI 방식**: 프랑스어 잘하는 사람에게 내가 하고 싶은 말을 전달하면, 그 사람이 대신 써줍니다.

AI 코딩도 똑같습니다. 여러분은 "사람 말"(원하는 것)을 하고, AI는 "코드 말"(만드는 방법)을 합니다. Claude Code는 아이디어와 실제 작동하는 소프트웨어 사이의 통역사입니다.

---

## AI 코딩이란?

예전에는 코드를 직접 한 줄씩 작성해야 했습니다. 문법을 외우고, 에러를 수정하고, 검색하고... 학습에 수년이 걸렸습니다.

지금은 다릅니다.

**AI에게 "이런 것을 만들어달라"고 말하면, AI가 코드를 작성합니다.**

### 왜 이것이 가능합니까?

Claude 같은 LLM(대규모 언어 모델)은 인터넷의 수많은 코드를 학습했습니다. GitHub의 오픈소스, Stack Overflow의 답변, 기술 블로그 등 수십억 줄의 코드를 봤습니다.

따라서 "파란색 제목이 있는 웹페이지"라고 말하면 다음과 같이 동작합니다:
- "파란색 제목" → `color: blue` 패턴
- "웹페이지" → HTML 구조 패턴

이렇게 학습한 패턴을 조합해서 코드를 생성합니다. 마법이 아니라 **패턴 매칭**입니다.

물론 아무렇게나 말하면 안 되고, 잘 표현해야 잘 만들어줍니다. 이 커리큘럼이 필요한 이유입니다.

### 예시: 간단한 웹페이지 만들기

**예전 방식:**
```html
<!DOCTYPE html>
<html>
<head>
    <title>내 페이지</title>
    <style>
        body { font-family: sans-serif; }
        h1 { color: blue; }
    </style>
</head>
<body>
    <h1>안녕하세요</h1>
    <p>제 이름은 홍길동입니다.</p>
</body>
</html>
```
이 모든 것을 외워서 직접 작성해야 했습니다.

**AI 코딩 방식:**
```
"파란색 제목으로 '안녕하세요'가 나오고,
그 아래 '제 이름은 홍길동입니다'가 나오는 웹페이지 만들어줘"
```
LLM은 이런 요청을 받으면 학습한 HTML/CSS 패턴 중에서 가장 적합한 것을 조합합니다. 웹페이지 구조(`<html>`, `<body>`), 제목 스타일(`color: blue`), 텍스트 배치 등을 이미 알고 있기 때문입니다.

## Vibecoding이란?

**Vibecoding**은 AI와 대화하면서 코드를 만드는 새로운 방식입니다.

"Vibe"는 느낌, 분위기라는 뜻입니다. 정확한 문법을 몰라도, 원하는 "느낌"을 전달하면 AI가 이해하고 구현한다는 의미입니다.

### Vibecoding의 핵심

1. **의도 전달**: "버튼 클릭하면 숫자가 올라가게 해줘"
2. **대화하며 수정**: "숫자가 너무 작아. 크게 해줘"
3. **협업**: AI가 제안하고, 사용자가 선택하고, 다시 AI가 수정

기존 코딩은 혼자 모든 것을 알아야 했습니다. Vibecoding은 AI라는 파트너와 함께 만듭니다.

## Claude Code란?

**Claude Code**는 Anthropic이 만든 AI 코딩 도구입니다.

### 웹 채팅(ChatGPT, Claude.ai)과의 차이

ChatGPT나 Claude.ai 같은 웹 채팅에서도 코드를 생성할 수 있습니다. 하지만 불편합니다:

| 웹 채팅 | Claude Code |
|---------|-------------|
| 코드를 복사해서 파일에 붙여넣어야 함 | 파일을 직접 생성/수정 |
| 로컬 파일 접근 불가 | 프로젝트 전체를 파악 |
| 명령어 실행을 직접 해야 함 | 명령어를 대신 실행 |
| 매번 상황 설명 필요 | 현재 상황을 인지 |

**요약**: 웹 채팅은 "레시피를 알려주는 사람", Claude Code는 "직접 요리하는 셰프"입니다.

### 다른 AI 코딩 도구들

Claude Code만 있는 게 아닙니다. 인기 있는 옵션들:

| 도구 | 유형 | 설명 |
|------|------|------|
| **Claude Code** | CLI (터미널) | Anthropic 공식 CLI. 모든 터미널에서 작동 |
| **Cursor** | IDE | VS Code 기반 AI 내장 에디터 |
| **Windsurf** | IDE | Codeium의 AI 중심 에디터 |
| **GitHub Copilot** | 확장기능 | VS Code, JetBrains 등의 AI 어시스턴트 |

**왜 Claude Code를 배웁니까?**
- 모든 터미널에서 작동, IDE에 종속되지 않음
- 파일과 명령어에 직접 접근
- 이미 쓰는 에디터와 함께 사용 가능
- CLI 기반 AI를 이해하면 모든 도구에 도움됨

이 커리큘럼은 Claude Code에 집중하지만, 개념은 다른 AI 도구에도 적용됩니다.

### Claude Code로 할 수 있는 것

- 웹사이트 만들기
- 앱 만들기
- 파일 정리 자동화
- 데이터 처리
- 기존 코드 수정
- 버그 찾아서 수정

## 학습 성과

24개 챕터를 완료하면 다음을 할 수 있습니다:

- **포트폴리오 웹사이트** 제작 및 배포
- **간단한 앱** (TODO 앱, 게임 등) 개발
- **자동화 도구** (파일 정리, 데이터 처리) 구축
- **AI 서비스** (문서 요약기, 번역기) 구현

코딩을 "배우는" 것이 아니라, **AI와 함께 "만드는" 것**을 학습합니다.

---

## 시작 전 마음가짐

### 1. 완벽하지 않아도 됩니다
처음에는 이상한 결과가 나올 수 있습니다. 다시 말하면 됩니다.

### 2. 에러는 당연합니다
코딩에서 에러는 실패가 아니라 과정입니다. AI가 에러도 수정해줍니다.

### 3. 작게 시작하십시오
처음부터 대단한 것을 만들려고 하지 마십시오. "Hello World"부터 시작합니다.

### 4. 궁금하면 물어보십시오
"이게 뭐야?", "왜 이렇게 해?" AI에게 물어보면 설명해줍니다.

---

## 따라해보세요 (미리보기)

아직 Claude Code를 설치하지 않았지만, 대화가 어떻게 이루어지는지 미리 살펴보겠습니다:

**예시 1: 웹페이지 만들기**
```
나: 내 이름 "김철수"를 큰 파란 글씨로 보여주는 웹페이지 만들어줘

Claude: 만들어드릴게요...
[index.html 파일을 스타일과 함께 생성]
```

**예시 2: 작업 자동화**
```
나: 이 폴더의 .jpeg 파일들 전부 .jpg로 이름 바꿔줘

Claude: 파일 이름을 변경할게요...
[모든 파일을 자동으로 변경]
```

**예시 3: 코드 수정**
```
나: 이 코드가 안 돌아가. 고쳐줘.
[문제 있는 코드 붙여넣기]

Claude: 문제를 찾았어요. 에러 원인은...
[설명하고 코드 수정]
```

보이십니까? 코딩 지식이 필요 없습니다. 원하는 것을 평범한 말로 설명하면 됩니다.

## 자주 하는 실수 (시작하기 전에)

### 1. 코딩부터 배워야 한다고 생각하기
그럴 필요 없습니다. 그것이 바로 핵심입니다. Claude Code로 시작하면, 하다 보면 자연스럽게 코딩 개념을 익히게 됩니다.

### 2. 첫 번째 결과가 완벽할 거라고 기대하기
AI는 똑똑하지만 독심술사는 아닙니다. 첫 결과가 원하는 게 아니면, "그게 아니라, 내 말은..." 하고 다시 말하면 됩니다.

### 3. 너무 막연하게 요청하기
"멋진 거 만들어줘"는 AI가 해석하기 어렵습니다. "클릭하면이라고 쓰인 파란색 버튼 만들어줘"가 훨씬 명확합니다.

### 4. 에러 한 번에 포기하기
에러는 전문 개발자에게도 일어납니다. 뭔가 잘못되면, Claude에게 에러를 말하고 도움을 요청하세요.

---

## 안 되면?

**너무 복잡하게 느껴지십니까?**
정상입니다. 한 챕터씩 천천히 진행하시기 바랍니다. 모든 것을 바로 이해할 필요는 없습니다.

**본인에게 맞을지 불안하십니까?**
이메일을 쓸 수 있으면 Claude Code도 사용할 수 있습니다. 대화하는 것과 같습니다.

**무언가 망가뜨릴까 걱정되십니까?**
Claude Code에는 안전 기능이 있습니다. 변경하기 전에 허락을 구하며, 언제든 거절할 수 있습니다.

**시작 전에 질문이 있으십니까?**
좋습니다. 호기심이 최고의 도구입니다. 질문을 적어두고 배우면서 Claude에게 물어보시기 바랍니다.
