## 기본 대화 방법

Claude Code는 채팅처럼 대화합니다. `>` 뒤에 입력하면 됩니다.

### 대화 시작

```bash
# 터미널에서 Claude Code 시작
claude
```

시작하면 다음 화면이 나타납니다:

```
>
```

여기에 입력하면 됩니다.

### 예시

```
> 안녕!
```

```
> 오늘 날씨 어때?
```

```
> 이 폴더에 뭐가 있어?
```

무엇이든 질문할 수 있습니다. 한국어도 잘 이해합니다.

## 권한 모드 (필수 이해)

Claude Code는 세 가지 모드가 있습니다. **이것이 가장 중요합니다.**

### 왜 권한 관리가 필요한가요?

LLM은 여러분의 요청을 "해석"해서 실행합니다. 대부분 정확하지만, 가끔 의도와 다르게 해석할 수 있습니다.

예를 들어 "이 폴더 정리해줘"라고 하면:
- 파일을 카테고리별로 분류하는 것일 수도 있고
- 불필요한 파일을 삭제하는 것일 수도 있습니다

LLM은 맥락에서 가장 적절한 해석을 선택하지만, 100% 확신할 수는 없습니다. 그래서 중요한 작업 전에 확인을 받는 것이 안전합니다.

### 1. Plan 모드 (가장 안전)

```
📋 Plan 모드
읽기만 가능. 아무것도 변경할 수 없음.
```

**사용 시점:**
- 처음 프로젝트를 살펴볼 때
- 코드 동작 방식을 이해하고 싶을 때
- 변경 전 계획을 세울 때

**진입 방법:**
```
> /plan
```
또는 `Shift + Tab` 두 번

### 2. Normal 모드 (기본)

```
🔒 Normal 모드
변경 전에 매번 확인을 요청함.
```

**사용 시점:**
- Claude Code를 처음 배울 때
- 중요한 파일을 다룰 때
- 변경 사항을 하나씩 확인하고 싶을 때

Claude가 파일을 수정하려고 하면:
```
Claude wants to edit src/app.js
[Allow] [Deny] [Allow all]
```
이렇게 확인을 요청합니다. Allow를 선택하면 수정됩니다.

### 3. Accept Edits 모드 (빠름)

```
⚡ Accept Edits 모드
파일 수정은 자동 승인. 위험한 명령만 확인 요청.
```

**사용 시점:**
- 이미 익숙해졌을 때
- 빠르게 작업하고 싶을 때
- Claude를 신뢰할 수 있을 때

### 모드 전환

| 방법 | 설명 |
|------|------|
| `Shift + Tab` | 모드 순환 (Normal → Accept Edits → Plan → Normal) |
| `/plan` | Plan 모드로 전환 |
| `claude --permission-mode plan` | 시작할 때 Plan 모드로 |

### Pro Tip

> "거의 항상 Plan 모드로 시작하세요."

새 작업을 할 때는 `Shift + Tab`을 두 번 눌러서 Plan 모드로 먼저 진입하세요. LLM은 전체 맥락을 파악한 상태에서 더 나은 코드를 생성합니다. 먼저 코드를 분석하고 계획을 세운 뒤 실행하면, "의도와 다르게 해석해서 다시 해야 하는" 상황을 줄일 수 있습니다.

## 자주 사용하는 단축키

외울 필요 없습니다. 필요할 때 확인하면 됩니다.

| 단축키 | 기능 |
|--------|------|
| `Shift + Tab` | 권한 모드 전환 |
| `Ctrl + C` | 현재 작업 취소 |
| `Esc Esc` | 되돌리기 (마지막 상태로) |
| `Ctrl + L` | 화면 정리 (대화는 유지) |
| `Shift + Enter` | 줄바꿈 (긴 메시지 작성 시) |

### Mac 사용자 참고

Mac에서 `Alt` 키 조합을 사용하려면 터미널 설정에서 "Option as Meta"를 활성화해야 합니다.

## 슬래시 명령어

`/`로 시작하는 특수 명령어입니다.

| 명령어 | 설명 |
|--------|------|
| `/help` | 모든 명령어 보기 |
| `/clear` | 대화 내용 삭제 |
| `/exit` | Claude Code 종료 |
| `/cost` | 현재까지 사용 비용 보기 |
| `/model` | 사용 중인 모델 보기/변경 |

### /clear vs Ctrl+L

- `/clear`: 대화 내용이 실제로 삭제됨. Claude가 이전 작업을 잊음.
- `Ctrl+L`: 화면만 정리됨. Claude는 기억을 유지함.

## 세션 이어서 하기

Claude Code를 종료했다가 다시 시작해도 이전 대화를 이어서 할 수 있습니다.

```bash
# 마지막 대화 이어서
claude --continue

# 과거 대화 중에서 선택
claude --resume

# 현재 세션에 이름 부여 (나중에 찾기 쉽게)
/rename my-project-work
```

## 실습

1. Claude Code 시작: `claude`
2. Plan 모드로 전환: `Shift + Tab` 두 번 (또는 `/plan`)
3. 현재 폴더 살펴보기:
   ```
   > 이 폴더에 뭐가 있어? 설명해줘.
   ```
4. Normal 모드로 전환: `Shift + Tab`
5. 간단한 파일 생성:
   ```
   > test.txt 파일 만들어줘. "Hello World"라고 써줘.
   ```
6. 승인 요청이 오면 `y` 또는 `Allow` 선택
7. 결과 확인:
   ```
   > 방금 만든 파일 내용 보여줘.
   ```

## 심화: 효과적인 Claude Code 활용

### 생각 → 계획 → 실행

AI 도구를 처음 접하면 바로 무언가를 입력하고 싶어집니다. 하지만 잠깐 멈추고 "내가 뭘 만들려고 하지?"를 먼저 정리하면 결과가 훨씬 좋아집니다.

Plan Mode는 이 과정을 돕습니다. Claude가 코드를 분석하고 접근 방법을 제안하면, 여러분은 그 계획이 맞는지 확인한 후 진행할 수 있습니다. 처음에 방향을 잘 잡으면 나중에 "다시 해야 하나..."라는 상황을 피할 수 있습니다.

### 권한 모드 선택 기준

어떤 모드를 선택할지 고민된다면, 두 가지를 생각해보세요:

**1. 작업의 위험도**
- 새 파일 생성 → 낮음
- 기존 코드 수정 → 중간
- 설정 파일이나 DB 관련 → 높음

**2. 되돌릴 수 있는가?**
- Git으로 추적 중 → 쉽게 되돌림
- 외부 서비스 호출 → 되돌리기 어려움

| 상황 | 권장 모드 |
|-----|---------|
| 위험 낮음 + 되돌리기 쉬움 | Accept Edits |
| 위험 중간 또는 처음 하는 작업 | Normal |
| 위험 높음 또는 되돌리기 어려움 | Plan |

### 명확한 요청이 명확한 결과를 만듭니다

"웹사이트 만들어줘"라고 하면 Claude는 수많은 가능성 중 하나를 선택해야 합니다. 반면 "포트폴리오 페이지 만들어줘. 소개, 프로젝트 3개, 연락처 섹션으로 구성하고, 다크 테마로"라고 하면 Claude가 정확히 원하는 것을 파악할 수 있습니다.

요청할 때 생각해볼 것:
- **무엇을**: 어떤 결과물을 원하는지
- **어디에**: 어떤 파일/위치에
- **어떻게**: 특별히 원하는 방식이 있다면
- **왜**: 맥락을 알면 더 나은 판단 가능

### 설계 결정은 함께

복잡한 기능을 만들 때는 바로 구현에 들어가기보다, Plan Mode에서 Claude와 구조를 논의하는 것이 좋습니다.

```
> /plan
> 사용자 인증 기능을 추가하려고 해.
> 어떤 방식들이 있고, 이 프로젝트에는 뭐가 적합할까?
```

Claude가 JWT, 세션, OAuth 등 옵션을 설명하면, 여러분이 선택하고, 그 다음 구현을 진행합니다. 이렇게 하면 "만들고 보니 요구사항과 맞지 않네"라는 상황을 피할 수 있습니다.
